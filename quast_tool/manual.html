QUAST 1.1 manual

QUAST stands for QUality ASsesment Tool. QUAST evaluates genome assemblies by computing various metrics.
This manual will help you to run the tool and understand its output.
You can find all project news and the latest versions of source code and manual at http://sourceforge.net/projects/quast/.
QUAST uses GeneMarkS, MUMmer and GAGE. All their sources are embedded in QUAST and you needn't to install them separately.

QUAST version 1.1 released under GPL v2 (see LICENSE for details) on 14 September 2012.

1. Installation

QUAST can be run on Linux and Mac OS.
It requires:

  • Python v. 2 (2.5 or higher)
  • Perl v. 5.6.0 or higher
It is highly recommended to install Matplotlib Python library for drawing plots.
It can be done on Ubuntu by typing: 

    sudo apt-get install python-matplotlib
Or with Python pip-installer: 

    pip install matplotlib
Or with Easy Install python module: 

    easy_install matplotlib
If you use Python 2.5, you also need to install simplejson (it is included in Python distribution starting from version 2.6): 

    apt-get install python-simplejson
or 

    pip install simplejson
or 

    easy_install simplejson

To download the source code tarball and extract it: 


    wget https://downloads.sourceforge.net/project/quast/quast-1.1.tar.gz
    tar -xzf quast-1.1.tar.gz
    cd quast-1.1

2. Running QUAST

2.1 Input data

QUAST can work with assemblies and references in FASTA format. Files may be compressed with gzip.

You can specify files with annotated genes and operons. We support the 2nd and the 3rd version of the GFF format, the format used by NCBI to store genes ("Summary (text)" format), and a plain TXT format with 4 tab-separated columns:

  • chromosome name (it has to match a name in the reference file),
  • gene or operon id,
  • start position,
  • end postion.
Coordinates are 1-based, i.e. the first nucleotide in the reference has position 1, not 0. If a start postion less than a corresponding end postion, such gene or operon is on a positive strand, otherwise it is on a negative strand.

You can find examples of a reference file and annotated genes and operons in all supported formats in your QUAST installation directory, test_data subdirectory. These files correspond to the first 10 Kbp of E. coli. Input reads and sample assemblies for this data set are also there.

2.2 GAGE mode

GAGE is one of the well-known assessment tools. However, it has issues that limit its use. For example, you can specify only one assembly per run, thereby it is difficult to compare a lot of assemblies simultaneously. In addition, GAGE has fixed the minimum contig length threshold (200 bp). All these issues are solved by QUAST in a GAGE mode. In this mode you can specify an unlimited number of assemblies and a minimum contig threshold as well. QUAST runs GAGE on all your assemblies with a specified contig threshold and creates files with values of the GAGE metrics (see GAGE site and GAGE paper for descriptions) in addition to a standard QUAST report.

Note: GAGE can work only with a reference genome.

2.3 Command line options

QUAST runs from a command line as follows: 


    python quast.py [options] <contig_file(s)>

QUAST options:

-o <output_dir>

    Specify an output directory. The default value is quast_results/results_<date_time>.

-R <filename>

    Reference genome file. Most metrics can't be evaluated without a reference.

-G <filename> (or --genes <filename>)

    Annotated genes file. See details about the file format in section 2.1

-O <filename> (or --operons <filename>)

    Annotated operons file. See details about the file format in section 2.1

--min-contig <int>

    Sets a lower threshold for a contig length. Only contigs of length greater than the threshold will be taken into account (except for some metrics, see section 3). The default value is 200.

--gage

    This flag starts QUAST in "GAGE mode" (see section 2.2). Note: in this case, you must also set -R option and you can set --min-contig option.

--contig-thresholds <int,int,...>

    Comma-separated list of contig length thresholds. Used in # contigs ≥ x and total length (≥ x) metrics (see section 3). The default value is 500,1000.

--genemark-thresholds <int,int,...>

    Comma-separated list of threshold lengths of genes to search with GeneMark. The default value is 0,300,1500,3000.

--not-circular

    This flag should be set if a genome is not circular (e.g., it is an eukaryote).

-h (or --help)

    Print help.

3. QUAST output

If an output path was not specified manually, QUAST puts its output into quast_results/result_<DATE>/ directory and creates a latest symlink to this folder inside quast_results/.

Output directory contents:
report.txt an assessment summary in a simple text format.
report.tsv a tab-separated version of the summary, suitable for Google Docs and Excel.
plots.pdf plots (file is created if the matplotlib python library is installed).
report.html an HTML version of the report with all plots inside it.

Note:

  • metrics based on a reference genome are computed only if the reference is given (see section 2.3),
  • metrics based on genes and operons are computed only if proper annotations are given (see section 2.3),

3.1 Metrics description

# contigs ≥ x  is total number of contigs of length ≥ x. This metric doesn't depend on --min-contig command line parameter (see section 2.3).

Total length (≥ x)  is the total number of bases in contigs of length ≥ x. This metric doesn't depend on --min-contig command line parameter (see section 2.3).


All remaining metrics are computed on the contigs of length ≥ x where x is specified in command line option --min-contig (see section 2.3, default is 200).

N50  is the contig length such that using longer or equal length contigs produces half (50%) the bases of the assembly. Usually there is no value that produces exactly 50%, so the more technical definition is the minimal length x such that using contigs of length at least x accounts for at least 50% of the total assembly length.

NG50  is the contig length such that using longer or equal length contigs produces half (50%) the bases of the reference genome. This metric could be computed only if the reference is given.

N75 and NG75  are defined similarly with 75% instead of 50%.

Number of contigs  is the total number of contigs in the assembly.

Largest contig  is the length of the longest contig in the assembly.

Total length  is the total number of bases in the assembly.

GC %  is the total number of G and C nucleotides in the assembly, divided by the total length of the assembly.

Reference length  is the total number of bases in the reference.

Reference GC %  is the total number of G and C nucleotides in the reference, divided by the total length of the reference.

Average %IDY  is the average of alignment identity percent (i.e. alignment accuracy) among all contigs.

# misassemblies  gives the number of positions at the assembled contigs where the left flanking sequence aligns over 1kb away from the right flanking sequence on the reference or they overlap on more than 1 kb or flanking sequences align on different strands. This metric could be computed only if the reference is given.

# misassembled contigs  is the number of contigs that contain misassembly events.

Misassembled contigs length  is the number of total bases contained in all contigs that have one or more misassemblies.

# unaligned contigs  is the number of contigs that have no alignment to the reference sequence. The value "X + Y part" means: X totally unaligned contigs plus Y partially unaligned contigs.

Unaligned contigs length  is the total number of unaligned bases in totally and partially unaligned contigs (only bases from unaligned parts).

# ambiguous contigs  is the number of contigs which have reference alignments of equal quality in multiple locations on the reference.

Ambiguous contigs length  is the number of total bases contained in all ambiguous contigs.

NA50, NGA50, NA75, NGA75 (A stands for "aligned")  like the same metrics without A but in this case we count lengths of aligned blocks instead of contigs lengths. I.e. if a contig has a misassembly with respect to the reference, we break the contig into smaller pieces. Also, if a contig has bases that don't align to the reference, they are not counted in NAx, NGAx.

Genome fraction (%)  is the total number of aligned bases in the reference, divided by the genome size. A base in the genome is counted as aligned if there is at least one contig with at least one alignment with this base. Repeates that map to multiple places may be counted many times in this quantity.

# genes  is the number of genes in the assembly (complete and partial), based on an annotated list of genes positions in the reference genome. The value "X + Y part" means X complete genes and Y partial ones (at least 100 bases are aligned but not the whole gene). This metric could be computed only if the reference and genes annotations file are given (see section 2.3).

# operons  is defined similarly to the previous metric but you need operons annotations file for computing this metric.

3.2 Plots descriptions

Cumulative length

    This plot shows the growth rate of assemblies lengthes. The y-axis is the size of assembly and x-axis is the number of contigs in the assembly (from the largest one to the smallest one).

Nx

    This plot shows the changes of Nx metric value in dependence of x value. The y-axis is the Nx value and x-axis is the x value.

NGx

    This plot is similar to the previous one but for NGx metric.

GC content plot

    This plot shows a distribution of percent of GC among contigs. The x value shows the percent of GC (from 0 to 100) and the y value show the total number of bp in contigs with such GC content. Typically, the distribution is similar to the Gaussian distribution. However, in case of contaminants with different GC content the distribution can have more than one curve.

Cumulative length (aligned contigs)

    This plot is similar to Cumulative length plot but we plot only aligned contigs instead of contigs itself. If a contig has a misassembly with respect to the reference, we break the contig into smaller pieces. This plot could be created only if the reference is given.

NAx and NGAx

    These plot are similar to the Nx and NGx plots but for NAx and NGAx metrics respectively. These plots could be created only only if the reference is given.

Genes

    This plot shows the growth rate of full genes in assemblies. The y-axis is the number of full genes in the assembly and x-axis is the number of contigs in the assembly (from the largest one to the smallest one). This plot could be created only if the reference and genes annotations file are given.

Operons

    This plot is similar to the previous one but for operons.
